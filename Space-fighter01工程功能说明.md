# Space-fighter01 工程功能说明

## 项目概述
Space-fighter01 是一个基于 HarmonyOS ArkUI 框架开发的跨设备飞行射击游戏应用，支持手机和平板设备。该项目展示了 HarmonyOS 应用接续（Continuation）能力的实际应用。

## 主要功能特性

### 1. 游戏核心功能
- **双游戏模式**：
  - **关卡模式**：逐关挑战，难度递增，每关需要击败特定数量的敌人
  - **无尽模式**：挑战生存极限，持续战斗获得最高分
- **实时游戏系统**：
  - 玩家战机控制（触摸移动）
  - 自动射击系统
  - 三种不同类型的敌人（低/中/高等级）
  - 碰撞检测和伤害系统
  - 粒子特效系统（爆炸效果）

### 2. 跨设备数据同步
- **分布式数据存储**：使用 HarmonyOS 分布式 KV 存储实现游戏数据跨设备同步
- **游戏记录保存**：
  - 最高得分
  - 最高关卡
  - 累计击毁敌人数
  - 按类型统计的击毁数
  - 无尽模式最佳分数
- **自动数据合并**：当用户在不同设备上登录同一华为账号时，游戏数据自动同步

### 3. 应用接续能力
- **多设备体验**：支持在手机和平板间无缝切换游戏
- **状态保持**：游戏进度和状态可在设备间迁移
- **统一用户界面**：针对不同设备尺寸进行适配

## 技术架构

### 项目结构
```
Space-fighter01/
├── AppScope/                    # 应用全局资源
├── features/                    # 功能模块
│   └── continue/               # 游戏核心功能模块
│       ├── src/main/ets/
│       │   ├── pages/          # 页面组件
│       │   │   ├── ContinueIndex.ets      # 导航容器
│       │   │   ├── GameModePage.ets       # 游戏模式选择页
│       │   │   └── GamePage.ets           # 游戏主页面
│       │   └── utils/
│       │       └── GameDataSync.ets       # 数据同步服务
│       └── build-profile.json5
├── product/                     # 产品模块
│   ├── phone/                  # 手机版本
│   └── tablet/                 # 平板版本
```

### 核心技术栈
- **开发框架**：HarmonyOS ArkUI
- **编程语言**：ArkTS
- **数据存储**：分布式 KV 存储 (distributedKVStore)
- **设备适配**：响应式 UI 设计
- **游戏引擎**：Canvas 2D 渲染 + 定时器游戏循环

## 核心组件说明

### 1. GameModePage (游戏模式选择页)
- 提供关卡模式和无尽模式选择
- 显示历史游戏记录
- 初始化游戏状态数据

### 2. GamePage (游戏主页面)
- 实现完整的游戏逻辑
- 包含：
  - 玩家战机控制
  - 敌人生成和移动
  - 子弹发射和碰撞检测
  - 生命值系统和游戏结束判定
  - 分数和关卡显示

### 3. GameDataSync (数据同步服务)
- 封装分布式 KV 存储操作
- 提供游戏记录的保存和加载
- 支持数据合并策略（取最大值或累加）

### 4. ContinueIndex (导航容器)
- 管理页面导航栈
- 处理页面间的切换逻辑

## 游戏机制

### 敌人系统
- **低级敌人**：1点生命值，1点伤害，20像素大小，绿色，1分
- **中级敌人**：2点生命值，1点伤害，28像素大小，黄色，2分  
- **高级敌人**：3点生命值，2点伤害，36像素大小，红色，3分

### 游戏参数
- 玩家生命值：5点
- 玩家攻击力：1点
- 子弹速度：8像素/帧
- 敌人基础速度：1.5像素/帧
- 无敌时间：1000毫秒
- 射击冷却：300毫秒
- 游戏循环：约60帧/秒
